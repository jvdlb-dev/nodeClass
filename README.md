# 🚀 Curso de Node.js

Repositório criado para concentrar os códigos, anotações, desafios e testes práticos desenvolvidos ao longo da trilha de **Node.js**. O objetivo é evoluir do básico de APIs sem frameworks até arquiteturas complexas, robustas e escaláveis no ecossistema backend.

## 🗺️ Grade Curricular e Evolução do Projeto

### 📂 Nível 1: Fundamentos do Node.js
Foco em entender a tecnologia por trás do ecossistema, construindo uma API RESTful do zero absoluto, sem bibliotecas externas.
- [x] Estrutura interna do Node.js, requisições, respostas e Headers.
- [x] Manipulação de dados e fluxos assíncronos com **HTTP e Streams nativas**.
- [ ] Diferentes tipos de parâmetros (Query, Route e Body parameters).
- **Desafio Prático:** 🛠️ *CRUD de Tarefas* (Gerenciamento de tasks local).

### 📂 Nível 2: Rotas e HTTP
Transição para ferramentas modernas e padrões de mercado utilizados no dia a dia.
- [ ] Construção de APIs REST utilizando o framework **Fastify** e **TypeScript**.
- [ ] Validação de dados de entrada e esquemas com **Zod**.
- [ ] Configuração de banco de dados SQL, migrações e queries com o query builder **Knex**.
- [ ] Tratamento de Cookies, variáveis de ambiente (`.env`) e testes automatizados.
- [ ] Deploy da aplicação em produção no **Render**.
- **Desafio Prático:** 🛠️ *Daily Diet API* (Controle diário de dieta).

### 📂 Nível 3: Implementando o SOLID
Aplicação de padrões de design para criar sistemas testáveis e fracamente acoplados.
- [ ] Desenvolvimento de uma API para check-ins em academias usando os princípios do **SOLID** e **Design Patterns**.
- [ ] Integração com banco de dados em ambiente de desenvolvimento via **Docker**.
- [ ] Autenticação segura com **JWT** e **Refresh Token**.
- [ ] Controle de acesso baseado em papéis (**RBAC**).
- **Desafio Prático:** 🛠️ *FindAFriend API* (Plataforma para adoção de animais).

### 📂 Nível 4: Domain-Driven Design (DDD)
Entendimento de como desenhar arquiteturas de software complexas guiadas pelas regras de negócio.
- [ ] Conceitos fundamentais de **DDD** e **Clean Architecture** aplicados a um fórum de discussões.
- [ ] Criação de *Value Objects*, *Subdomínios*, *Relacionamentos* e *Domain Events*.
- [ ] Tratamento de erros de forma funcional e escrita de testes unitários isolados.

### 📂 Nível 5: NestJS
Domínio do framework opinativo mais robusto do mercado de Node.js corporativo.
- [ ] Arquitetura estruturada com Injeção de Dependências, Controllers, Providers e Services.
- [ ] Integração com o ORM moderno **Prisma**, validações com **Zod** e testes com **Vitest**.
- [ ] Integração entre os conceitos de DDD desenvolvidos no módulo anterior e a estrutura do NestJS.
- **Desafio Prático:** 🛠️ *FastFeet API* (Sistema de controle de encomendas de uma transportadora).

## 🛠️ Tecnologias e Conceitos Dominados

- **Runtime & Linguagens:** Node.js, TypeScript, JavaScript (ES Modules).
- **Protocolos & Padrões:** HTTP/REST, Streams, SOLID, DDD, Clean Architecture.
- **Frameworks & Ferramentas:** Fastify, NestJS, Knex, Prisma, Docker, Zod.
- **Testes:** Vitest.
